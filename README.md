# todo-cli

> 命令行待办事项管理工具 —— 轻量、零依赖、即开即用。

## 项目简介

`todo-cli` 是一个基于命令行的待办事项（Todo）管理工具，帮助你在终端中快速记录、查看、完成、修改和删除任务。它使用单个 Python 文件实现，数据以 JSON 格式持久化存储，无需安装任何第三方库，适合日常个人任务管理场景。

## 功能特性

| 命令 | 功能 |
|---|---|
| `add` | 添加新任务，自动分配编号并记录创建时间 |
| `list` | 查看全部任务，按编号排序，区分已完成 / 未完成状态 |
| `done` | 标记指定任务为已完成，防止重复标记 |
| `update` | 修改指定任务的内容，保留其他信息不变 |
| `delete` | 删除指定任务，支持编号不存在时的错误提示 |

- 📝 创建时间自动记录
- ✅ / ⬜ 图标区分完成与未完成状态
- 🔢 编号全局递增，删除后不复用，保持稳定性
- 📊 添加任务后即时显示当前任务总量
- ⚠️ 操作失败时自动回滚，保障数据一致性

## 使用方法

### 添加任务

```bash
# 格式
python todo.py add "任务内容"

# 示例
$ python todo.py add "完成周报"
✅ 任务 #1 已添加：完成周报
   📊 当前共有 1 个待办任务
```

### 查看所有任务

```bash
# 格式
python todo.py list

# 示例
$ python todo.py list

──────────────────────────────────────────────────
  📋 待办事项列表（共 3 项）
──────────────────────────────────────────────────
  ✅  #1    完成周报
         创建于 2026-07-01 10:47:19
  ⬜  #2    买菜
         创建于 2026-07-01 10:47:19
  ✅  #3    学习Python
         创建于 2026-07-01 10:47:19
──────────────────────────────────────────────────
```

### 标记任务完成

```bash
# 格式
python todo.py done <编号>

# 示例
$ python todo.py done 1
🎉 任务 #1 已标记为完成：完成周报
```

### 修改任务

```bash
# 格式
python todo.py update <编号> "新内容"

# 示例
$ python todo.py update 1 "完成周报并提交给经理"
✅ 任务 #1 已更新
   旧内容：完成周报
   新内容：完成周报并提交给经理
```

### 删除任务

```bash
# 格式
python todo.py delete <编号>

# 示例
$ python todo.py delete 2
✅ 任务 #2 已删除：买菜
```

## 项目结构

```
todo-cli/
├── todo.py        # 主程序，包含全部命令逻辑、参数解析和错误处理
└── todos.json     # 数据文件，以 JSON 数组形式持久化存储任务列表（自动生成）
```

| 文件 | 说明 |
|---|---|
| `todo.py` | 单文件实现，包含 `add` / `list` / `done` / `update` / `delete` 五个命令的完整逻辑，以及 JSON 读写、Windows 编码适配、使用说明打印等辅助模块 |
| `todos.json` | 任务数据持久化文件，每个任务记录 `id`（编号）、`content`（内容）、`created_at`（创建时间）、`done`（完成状态）四个字段，首次添加任务时自动创建 |

## 技术说明

- **纯标准库**：仅使用 `json`、`os`、`sys`、`datetime` 内置模块，无需 `pip install`
- **JSON 持久化**：任务数据以 UTF-8 编码、缩进格式化的 JSON 数组存储，方便直接查看和编辑
- **Windows 兼容**：启动时自动将 `stdout` 重包装为 UTF-8 编码，解决 Windows 控制台 GBK 编码下 emoji 输出报错问题
- **事务性写入**：每次写盘操作都有成功校验，写入失败时自动回滚内存中的修改，杜绝数据不一致
- **容错设计**：JSON 文件不存在或损坏时自动降级为空列表，不会因数据问题导致程序崩溃
